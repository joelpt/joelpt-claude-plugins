# Tip of the Day Plugin - Design Document

**Date**: 2026-03-05
**Status**: Approved

## Overview

A Claude Code plugin that delivers a "Claude Code intelligence briefing" at each new session start. Tips are generated by Claude from both local sources (installed plugins, skills, release notes) and online sources (Anthropic blog, community content). The plugin maintains a curated tip database that self-refreshes every 24 hours, with tips scored for relevance based on the user's work patterns.

## Architecture

```
Session Start (fast path)           Generation Pipeline (async, ~24h)
+---------------------+            +------------------------------+
| session-start.sh    |            | generate-tips.sh             |
| - Read JSON DB      |  triggers  | - Scan local plugins/skills  |
| - Pick random tip   | ---------> | - Fetch online sources       |
| - Output to stdout  |  (if stale | - Read user profile          |
| - Check thresholds  |   or low)  | - Invoke claude CLI to       |
+---------------------+            |   generate + curate tips     |
         |                         | - Write updated JSON DB      |
         v                         +------------------------------+
   +----------+
   | tips.json |  (shared state)
   +----------+
```

### Two Execution Paths

1. **Fast path (every session start)**: Read JSON, pick weighted-random unshown tip, output as system-reminder, check if generation needed. Must complete in <1 second.

2. **Generation pipeline (async, every ~24h)**: Discover sources, generate new tips via `claude` CLI, curate existing tips (deduplicate, retire stale, consolidate overlapping, adjust scores), write updated database. Runs in background.

## Data Model (tips.json)

```json
{
  "schema_version": 1,
  "tips": [
    {
      "id": "tip-<uuid-short>",
      "content": "Human-readable tip text",
      "category": "plugin|skill|workflow|feature|blog|community",
      "source": "local-scan|anthropic-blog|github|docs|...",
      "keywords": ["web", "scraping", "firecrawl"],
      "created_at": "2026-03-05T12:00:00Z",
      "shown_at": null,
      "show_count": 0,
      "value_score": 0.8,
      "user_relevance_score": 0.7
    }
  ],
  "metadata": {
    "last_generation_run": null,
    "last_curation_run": null,
    "total_tips_generated": 0,
    "total_tips_retired": 0,
    "user_profile_summary": ""
  }
}
```

## Session Start Flow

1. Read `tips.json` (create with empty tips array if missing)
2. Filter to unshown tips (fallback: least-recently-shown if all shown)
3. Weight selection by `value_score * user_relevance_score`
4. Output selected tip as system-reminder via stdout (JSON hookSpecificOutput format)
5. Mark tip as shown: update `shown_at`, increment `show_count`
6. Check trigger conditions:
   - Unshown tips < 5? -> trigger generation (async)
   - Last generation > 24h ago? -> trigger generation (async)
7. Exit 0

## Generation Pipeline Flow

Runs asynchronously (does not block session start).

### Phase 1: Discovery
- Scan `~/.claude/plugins/` for all `plugin.json` and `SKILL.md` files
- Read configurable online sources from `config/sources.json`
- Use `claude` CLI with web tools to fetch and summarize online content

### Phase 2: User Profiling
- Check for episodic-memory plugin; if available, search recent conversations
- Otherwise scan recent `.claude/projects/` transcripts
- Build a brief user profile summary (work patterns, domains, tools used)

### Phase 3: Tip Generation
- Invoke `claude -p` with `prompt-generate.md` containing:
  - Discovered local plugin/skill data
  - Online source summaries
  - Current tip database (to avoid duplicates)
  - User profile for relevance biasing
- Claude generates new tip candidates as structured JSON

### Phase 4: Curation
- Invoke `claude -p` with `prompt-curate.md` containing:
  - Full tip database (existing + newly generated)
  - User profile
- Claude performs:
  - Deduplication: merge overlapping tips into consolidated versions
  - Staleness removal: retire tips about outdated features/practices
  - Score adjustment: update value_score and user_relevance_score based on profile
  - Pool management: if total tips > max_tips, retire lowest-scored tips
- Write updated `tips.json`

## Skill (/totd)

User-invocable skill supporting:
- Show current tip of the day
- List recent tips
- Request tip on a specific topic
- Force regeneration
- Show tip stats (total tips, unshown count, last generation time)

## Presentation

Tips are injected as system-reminders via SessionStart hook stdout. The system-reminder instructs Claude to present the tip in a formatted block at conversation start.

### Future Enhancement (v2+)
- Keyword-triggered contextual tips: Claude "knows" all tips and surfaces relevant ones when keywords appear during conversation (ambient mode).

## Technology Stack

- **Bash** for all scripts (session-start.sh, generate-tips.sh)
- **`claude` CLI** for tip generation and curation (non-interactive mode with `-p`)
- **`jq`** for JSON manipulation in the fast path
- **No external dependencies** beyond what a Claude Code user already has

## Design Decisions

1. **Session start stays fast**: Only reads JSON + picks tip. No network calls, no Claude invocations on the hot path.
2. **Generation is always async**: Runs in background via hook's async capability, never blocks session start.
3. **Claude generates AND curates**: Uses `claude` CLI for both creation and maintenance, keeping quality high.
4. **User profiling is soft**: Influences scoring, doesn't hard-filter. Users still get diverse tips.
5. **Configurable sources**: Users can add their own blogs/sources via sources.json.
6. **24-hour refresh cycle**: Default staleness threshold reflects the fast-moving Claude Code ecosystem.
7. **JSON file storage**: Simple, no dependencies, sufficient for expected data volume (<1000 tips). Easy to refactor to SQLite later if needed.

## Trigger Conditions for Generation

| Condition | Threshold | Rationale |
|-----------|-----------|-----------|
| Unshown tips below watermark | < 5 | Ensures user always has fresh tips |
| Time since last generation | > 24 hours | Keeps tips current with fast-moving ecosystem |
| Both checked at session start | Async trigger | Never blocks the user |
